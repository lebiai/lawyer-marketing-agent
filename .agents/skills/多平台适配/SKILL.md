# 多平台适配

## 工作流

```
Step 1: 调用 adapt_platform 进行智能结构转换
Step 2: 输出适配后的文案 + 转换说明 + 优化建议
Step 3: 调用 store_knowledge(my_articles)
Step 4: 调用 log_conversation
```

## 转换效果

| 来源 → 目标 | 结构变化 |
|-------------|---------|
| 公众号 → 小红书 | 长文 → 痛点开头+分点干货+互动结尾 |
| 公众号 → 抖音 | 长文 → 黄金60秒（钩子→展开→亮点→引导） |
| 小红书 → 抖音 | 图文 → 口播脚本+分段时间 |
| 小红书 → 公众号 | 短文 → 导语+分层论述+金句 |
| 抖音 → 小红书 | 口播 → 图文分段+emoji+标签 |
| 抖音 → 公众号 | 口播 → 深度长文+小标题+配图建议 |

```

📌 [来源] → [目标]

📝 标题：[新标题]（X字 ✓/需调整）

📄 正文：
[适配后的完整文案]

转换说明：
✅ [结构重构说明]
✅ [标题调整说明]
✅ [格式适配说明]

💡 优化建议：
▸ [建议1]
▸ [建议2]
▸ [建议3]

🏷️ 标签：[标签列表]
════════════════════════════════
```

## ⚠️ 自查清单
- [ ] 标题是否符合目标平台的字数限制
- [ ] 结构是否按目标平台的格式重组
- [ ] 是否输出了转换说明和优化建议
- [ ] 是否调用了 store_knowledge
- [ ] 是否调用了 log_conversation
