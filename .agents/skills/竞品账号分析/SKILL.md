# 竞品账号分析

使用 blogger-distiller 数据分析引擎，通过 TikHub API 采集公开数据进行深度分析。

## ⚠️ 使用铁律

1. **必须提供账号链接**，不接受到名称搜索。用户不给链接就不工作。
2. **必须先告知费用**，用户确认后才执行。
3. **余额不足** → 引导用户联系 iodun001 充值。

## 工作流

```
Step 1: 检查权限 → check_tikhub_status
Step 2: 未开通 → 引导加微信。已开通 → 继续
Step 3: 要求用户提供账号链接
Step 4: parse_account_link(url) → 自动识别平台 + 提取 user_id
Step 5: 展示分析深度选项（采集数量+评论深度合并）：
        ① ⚡ 轻量 — 30条内容 + 20条评论/条（¥0.5~1.5，约15分钟）
        ② ⭐ 标准 — 50条内容 + 50条评论/条（¥1.0~3.0，约30分钟）
        ③ 🔬 深度 — 80条内容 + 100条评论/条（¥2.0~5.0，约45分钟）
Step 6: 用户选择 → analyze_account(user_id=xxx, max_notes=XX, max_comments=YY)
        → 自动检查余额 → 采集（并行 N 个 worker 加速）→ 合并 → 分析 → 报告 → 记账
Step 7: 展示报告 + 费用
Step 8: 提取风格 → store_knowledge(content_samples)
Step 9: store_knowledge(competitor_analysis) + log_conversation
```

## 并行架构

采集阶段使用多 worker 并行加速：
- `parallel_workers=3`（默认）：拆分笔记为 3 个 chunk，并行采集详情+评论
- 每多一个 worker，速度约提升 1.7~2×（取决于 API 限速）
- 流程：list-only(获取ID) → 拆分chunk → 并行采集 → 合并 → 分析

对于 50 条内容：顺序约 30-45 分钟 → 并行约 12-18 分钟

## 评论深度

每条内容默认采集 top 20 条热门评论（按点赞排序）。如果某个内容的评论数不足用户选择的数量，则全量采集，不做补调。

## 成本参考

| 采集数量 | 预计费用 | 耗时 |
|---------|---------|------|
| 30条 | ¥0.5~1.5 | 约15-25分钟 |
| 50条 | ¥1.0~3.0 | 约30-45分钟 |
| 80条 | ¥2.0~5.0 | 约45-65分钟 |

## 错误处理

| 错误 | 回复 |
|------|------|
| 没给链接 | "请提供小红书或抖音账号主页链接，不接受到名称搜索" |
| 余额不足 | "TikHub 账户余额不足，请联系微信 iodun001 充值" |
| 采集失败 | 显示错误信息，账单记录为失败 |

分析完成后，按以下顺序展示：

#### 1️⃣ 完整竞品分析报告
展示  字段全文（report.md），使用代码块或直接文本。

#### 2️⃣ 完整分析数据
展示  字段全文（analysis.json），包含：
- **stats** — 基础统计（总篇数、总点赞/收藏/评论、均值）
- **category_stats** — 内容分类分布（每类篇数、占比、均赞）
- **tag_freq** — 高频话题标签（排行）
- **top10** — 高互动内容TOP10（含标题、互动数据）
- **opinion_candidates** — 观点句提取（认知层）
- **value_words** — 高频价值词
- **writing_structure** — 写作结构分析
- **notes** — 每篇内容的原始数据（标题、正文、图片、互动、评论等）

## 报告复看

分析完成后，HTML 报告自动注册到报告注册表。用户随时可以说：
- "打开张三的分析报告"
- "看看之前分析的报告"

→ 调用 `open_report(account_name)` 查找并打开对应 HTML。
