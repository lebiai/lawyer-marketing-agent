# 竞品账号分析

使用 blogger-distiller 数据分析引擎，通过 TikHub API 采集公开数据进行深度分析。

## ⚠️ 使用铁律

1. **必须提供账号链接**，不接受到名称搜索。用户不给链接就不工作。
2. **必须先告知费用**，用户确认后才执行。
3. **余额不足** → 引导用户联系 iodun001 充值。

## 工作流

```
Step 1: 检查权限 → check_tikhub_status
Step 2: 未开通 → 回复加微信引导。已开通 → 继续
Step 3: 要求用户提供账号链接
        "请提供小红书或抖音账号主页链接"
Step 4: 调用 parse_account_link(url) 解析
        自动识别平台 + 提取 user_id
Step 5: 询问采集数量 30/50/80 条
Step 6: 调用 get_cost_estimate(max_notes) 获取费用估算
Step 7: 告知用户费用并确认
        "采集50条约 ¥1.0~3.0，确认继续？"
Step 8: 用户确认 → 调用 analyze_account(..., user_id=xxx, max_notes=50)
        → 自动检查余额 → 采集 → 分析 → 存报告 → 记账单
Step 9: 展示报告 + 费用明细
Step 10: 提取风格特征 → store_knowledge(content_samples)
Step 11: store_knowledge(competitor_analysis) + log_conversation
```

## 成本参考

| 采集数量 | 预计费用 | 耗时 |
|---------|---------|------|
| 30条 | ¥0.5~1.5 | 约15-25分钟 |
| 50条 | ¥1.0~3.0 | 约30-45分钟 |
| 80条 | ¥2.0~5.0 | 约45-65分钟 |

## 错误处理

| 错误 | 回复 |
|------|------|
| 没给链接 | "请提供小红书或抖音账号主页链接，不接受到名称搜索" |
| 余额不足 | "TikHub 账户余额不足，请联系微信 iodun001 充值" |
| 采集失败 | 显示具体错误信息，账单记录为失败状态 |
